# Step 04 - Report And Status

Write the report with `references/report-template.md`.

1. Fill metadata, including missing provenance as `Not evidenced`.
2. Write a concise executive summary from matrix counts and validation level.
3. Cite the initial trigger.
4. Summarize scope without rewriting ACs.
5. Summarize implementation by domain or story with anchors.
6. Insert the traceability matrix.
7. List completion evidence by type.
8. List validation results with scope.
9. List deviations, limits, assumptions, residual risks, and evidence gaps.
10. Recommend actions tied to a gap, risk, skipped check, or out-of-scope item.
11. Assign final status using workflow rules.
12. Persist to the default path unless the user requested chat-only output.
13. Apply the workflow overwrite/versioning rule when the path exists.

Do not update `story-status.md` unless the user explicitly asks for status
synchronization.
