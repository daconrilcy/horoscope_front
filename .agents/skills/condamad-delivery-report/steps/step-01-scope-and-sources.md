# Step 01 - Scope And Sources

Identify the delivery target before collecting evidence.

1. Locate repo root and git status when `.git` exists.
2. Resolve target story keys/range, branch/range, diff, or initiative.
3. Identify the source trigger: audit, bug, QA, review, spec, decision, or user
   request.
4. List available evidence: capsules, final evidence, review artifacts,
   validation logs/commands, diff/changed files, and source documents.
5. Prepare metadata: generated time, repo, branch, commit/diff source, covered
   stories, source documents, and validation source (`story-time`, `CI`,
   `report-time`, `external`, or `Not evidenced`).
6. Record missing expected inputs as potential `Evidence gaps`.

Stop only if no story, diff, source document, or implementation target can be
identified.
