# Story portable-llm-release-readiness: Rendre la readiness release LLM portable

Status: ready-for-review

## 1. Objective

Remplacer le chemin absolu local du cache pytest dans
`scripts/llm-release-readiness.ps1` par un chemin derive du repo root.
Conserver une option de surcharge explicite.

## 2. Trigger / Source

- Source type: audit
- Source reference: `_condamad/audits/scripts-ops/2026-05-02-1847/03-story-candidates.md` (`SC-006`)
- Source finding: `_condamad/audits/scripts-ops/2026-05-02-1847/02-finding-register.md` (`F-006`)
- Reason for change: le script release LLM encode `C:\dev\horoscope_front`, ce qui casse la portabilite entre clones.

## 3. Domain Boundary

This story belongs to one primary runtime domain and one directly required
supporting backend release domain:

- Domain: `scripts/llm-release-readiness.ps1`
- In scope:
  - Remplacer le cache pytest absolu par un chemin repo-relative.
  - Ajouter un parametre de surcharge explicite pour le cache.
  - Conserver les artefacts de preuve existants du script.
  - Adapter les chemins pytest obsoletes du script vers les fichiers collectes
    actifs couvrant le meme signal release.
  - Corriger les preconditions backend minimales necessaires pour que les tests
    release appeles par le script soient repetables: versions de snapshots
    uniques en integration et prechargement des relations deja serialisees par
    le snapshot.
- Out of scope:
  - Modifier les tests LLM executes par le script, sauf remplacement documente
    d'un chemin obsolète non collecte par son chemin actif equivalent.
  - Changer les criteres de release LLM.
  - Modifier les prompts, assemblies ou providers LLM.
- Explicit non-goals:
  - Ne pas changer `RG-016` a `RG-022` sur les invariants LLM.
  - Ne pas introduire de fallback vers `C:\dev\horoscope_front`.
  - Ne pas modifier les contrats API.
