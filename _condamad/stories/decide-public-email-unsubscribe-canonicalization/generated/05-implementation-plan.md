# Implementation Plan

## Findings

- `EmailService.get_unsubscribe_link` generates `/api/email/unsubscribe?token=...`.
- Onboarding templates consume `unsubscribe_url` generated by the email service.
- Integration tests call `/api/email/unsubscribe` directly.
- Prior audit evidence classifies the route as `external-active`.
- No explicit user approval exists to delete or migrate the public URL.

## Selected approach

- Select `needs-user-decision` and keep runtime behavior unchanged until explicit approval exists.
- Preserve the runtime path, handler behavior, generated links, and OpenAPI path.
- Update the exception register decision text so the route is no longer "undecided until migration story".
- Strengthen architecture tests so the decision remains exact and audited.

## Files to modify

- `backend/app/api/route_exceptions.py`
- `backend/app/tests/unit/test_api_router_architecture.py`
- story evidence artifacts under `_condamad/stories/decide-public-email-unsubscribe-canonicalization/`

- Unsubscribe consumer and duplicate handler scans.

## Rollback

- Revert the route exception decision text and the exact assertions if a future approved migration story selects a canonical replacement.
