# Story formalize-scripts-ownership: Formaliser l'ownership du dossier scripts

Status: ready-for-review

## 1. Objective

Creer une topologie canonique documentee pour `scripts/`.
Chaque script racine doit avoir un proprietaire, une famille, une commande de
validation et une decision de support explicites.

## 2. Trigger / Source

- Source type: audit
- Source reference: `_condamad/audits/scripts-ops/2026-05-02-1847/03-story-candidates.md` (`SC-003`)
- Source finding: `_condamad/audits/scripts-ops/2026-05-02-1847/02-finding-register.md` (`F-003`)
- Reason for change: le dossier `scripts/` est plat et melange qualite, securite, DB, perf, LLM, dev local et validateurs story sans ownership visible.

## 3. Domain Boundary

This story belongs to exactly one domain:

- Domain: `scripts/`
- In scope:
  - Inventorier tous les fichiers renvoyes par `rg --files scripts`.
  - Creer un registre canonique d'ownership pour les familles `quality`, `security`, `db`, `perf`, `llm`, `dev`, `story-tools`.
  - Ajouter un garde qui verifie qu'un nouveau script possede une ligne exacte dans ce registre.
- Out of scope:
  - Deplacer physiquement les scripts dans des sous-dossiers.
  - Supprimer `stripe-listen-webhook.sh`.
  - Modifier le comportement metier des scripts.
- Explicit non-goals:
  - Ne pas changer `RG-015`; cette story doit le renforcer pour les scripts.
  - Ne pas creer de chemins legacy paralleles ou de re-export de scripts.
  - Ne pas modifier les routes API, contrats OpenAPI, ou composants frontend.

## 4. Operation Contract

- Operation type: converge
- Primary archetype: namespace-convergence
- Archetype reason: la story converge un namespace plat vers une ownership canonique documentee avant tout deplacement futur.
- Behavior change allowed: no
- Behavior change constraints:
  - Les chemins executables des scripts restent inchanges.
  - Les commandes documentees continuent de pointer vers les fichiers actuels.
- Deletion allowed: no
- Replacement allowed: no
- User decision required if: la classification d'un script actif necessite une suppression ou un changement de support cross-platform.

Use for boundary, namespace, service, API, core, domain, or infra refactors.
