## Objective

Implement the canonical backend-domain builder that produces `beginner_summary_v1` from `structured_facts_v1`.

The implementation must reuse or extend the closest existing projection owner, produce deterministic B2C summary output, support normal, empty,
degraded and unavailable states, handle missing birth time without unsupported claims, and keep internal runtime primitives out of public payloads.

## Target State

- `beginner_summary_v1` is generated by one tested backend builder or a clearly extended existing projection owner.
- The builder consumes `structured_facts_v1` as its upstream factual source instead of recalculating astrology runtime data.
- The output is deterministic, JSON-serializable and free from LLM-generated narration.
- The output supports `normal`, `empty`, `degraded` and `unavailable` states with controlled message codes.
- Missing birth time produces a controlled degraded mode without ascendant or house-dependent claims.
- Static disclaimer references from CS-284 are attached or referenced without LLM-authored text.
- Internal primitives, raw runtime objects, debug traces, audit details and technical evidence are not exposed.
- No frontend source, public API route, database migration, premium projection or long narrative renderer is introduced.

## Contract Shape

- Contract type:
  - Backend-domain runtime builder payload for a deterministic beginner B2C projection.
- Fields:
  - `projection_id`: exact value `beginner_summary_v1`.
  - `source_projection_id`: exact value `structured_facts_v1`.
  - `state`: one of `normal`, `empty`, `degraded` or `unavailable`.
  - `summary_items`: deterministic client-readable facts for beginner users.
  - `main_signs`: deterministic sign highlights derived from structured facts.
  - `ascendant`: emitted only when birth time supports it.
  - `dominant_house`: emitted only when house data is available.
  - `dominant_themes`: deterministic theme identifiers or labels derived from structured facts.
  - `message_code`: stable code for client display.
  - `disclaimer_codes`: static application-controlled disclaimer references.
  - `missing_data`: controlled public reasons such as missing birth time.
  - `excluded_surfaces`: raw runtime objects, debug traces, audit details, evidence refs, prompts and LLM output.
- Required fields:
  - `projection_id`
  - `source_projection_id`
  - `state`
  - `summary_items`
  - `dominant_themes`
  - `message_code`
  - `disclaimer_codes`
  - `excluded_surfaces`
- Optional fields:
  - `ascendant` only when birth time supports it.
  - `dominant_house` only when house data is available.
  - `missing_data` only for `degraded` or `unavailable` states.
- Status codes:
  - none; no HTTP endpoint implementation is authorized.
- Serialization names:
  - emitted JSON keys must be stable snake_case names from the projection contract.
- Frontend type impact:
  - none; no frontend source or generated client is touched.
- Generated contract impact:
  - `app.openapi()` must not expose `beginner_summary_v1` from this story.
